# Игровой движок на JavaScript

## Описание
Это учебный проект по созданию 2D игрового движка на чистом JavaScript (ванильный JS) с использованием Webpack, SCSS (БЭМ). Проект включает:
- Базовый игровой цикл с `requestAnimationFrame` и `deltaTime`
- Управление персонажем с клавиатуры
- Гравитация, прыжки, платформы (односторонние и двусторонние)
- Столкновения (AABB)
- Менеджер сущностей
- Интерактивная панель управления для настройки параметров игрока, врагов, платформ в реальном времени
- Отображение FPS и счётчика столкновений

Цель проекта — освоить архитектуру игрового движка и подготовить основу для дальнейшего развития: редактор уровней, камера, тайловая карта, десктопная версия через Electron.

## Технологии
- **JavaScript (ES6+)** — вся логика
- **Webpack** — сборка, dev-сервер
- **SCSS** — стилизация (БЭМ)
- **HTML5 Canvas** — рендеринг

## 3. Анализ текущего кода и правки

Проведём ревизию на предмет ошибок, улучшений и соответствия архитектуре.

### 3.1. Ошибки и недочёты
- **В `EntityManager.update`** ранее мы добавили третий аргумент `allEntities`, но в `index.js` при вызове `entityManager.update(deltaSec, input.getState())` мы не передаём список. Это исправлено, так как в `EntityManager.update` мы сами формируем `allEntities` через `getAllEntities()`. Всё корректно.
- **В `Player.update`** при проверке коллизий по X используется `this.vx`, который может быть 0, тогда коррекция не срабатывает – это нормально.
- **Односторонние платформы** работают, но при быстром движении возможны провалы – это допустимо на данном этапе.
- **В `ControlPanel`** при изменении размеров игрока, позиция корректируется с `delta/2` для ширины и `-delta` для высоты – это может привести к смещению игрока относительно пола. Альтернатива – оставить без автоматической коррекции, а лишь ограничивать координаты. Пока оставим как есть.

### 3.2. Улучшения, которые стоит внести
- **Добавить обработку изменения размеров игрока с учётом пола**: при увеличении высоты игрок может провалиться под пол, нужно корректировать Y.
- **В `ControlPanel` обновлять `playerYSlider` при изменении Y игрока из других мест (например, при падении)** – можно добавить слушатель на изменение Y игрока, но проще обновлять слайдер при отрисовке (раз в кадр). В текущей реализации слайдер не синхронизируется с падением. Добавим обновление в `draw()` или в `update()` перед отрисовкой.
- **Переименовать `ControlPanel.init` в `constructor`** и вызывать логику внутри, но оставим как есть.
- **Вынести `createEnemyItem` и `createPlatformItem` как приватные методы** – уже сделано.
